# Project preview

This course is **not** primarily a deep dive into Dagster fundamentals. For that, we recommend [Dagster Essentials](https://courses.dagster.io/courses/dagster-essentials), where you'll learn assets, resources, schedules, partitions, and more from the ground up.

---

## What this course is about

**AI Driven Data Engineering** is about **jumpstarting your developer workflow** and making the most of the AI tools Dagster maintains. Using the right AI tools and patterns, you don't necessarily need to know every Dagster abstraction before you start building. You can:

- Generate and refine Dagster code with AI assistance
- Follow patterns that keep your code consistent and maintainable
- Get features to production quickly, then deepen your understanding as you go

As you get more comfortable, diving into Dagster Essentials (or specific docs) will help you understand *why* things are built the way they are and how to customize further.

---

## What's ahead

In the lessons that follow, we'll walk through the AI tool landscape, how Dagster's AI investments fit in, and how to use these tools to write better code and adhere to Dagster best practices—so you can build and ship data applications faster and with more confidence.
