# Overview: Managing AI sessions

Plan mode and subagents give you tools for handling complex tasks. Context management gives you habits for the everyday work between those tasks.

Every session accumulates history: the files you've read, the code you've written, the errors you've debugged, the decisions you've reversed. As that history grows, earlier details get compressed. The model doesn't forget suddenly—it degrades gradually, in ways that are observable if you know what to look for.

This lesson is about recognizing when a session is degrading and responding deliberately. That means:

- Understanding what fills a context window in a real working session
- Knowing the concrete signals that recall is getting imprecise
- Building habits that keep sessions scoped and productive
- Knowing when starting fresh is cleaner than continuing
- Carrying relevant state forward into a new session without losing ground

These aren't emergency procedures for when something goes wrong. They're operating habits for every session.
